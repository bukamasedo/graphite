# i18n

i18next + react-i18next を使用。言語は `en`（デフォルト）と `ja`。

## ディレクトリ構成

```
src/i18n/
├── index.ts            # i18n 初期化（en/ja リソース登録）
├── locales/
│   ├── en.json         # 英語キー
│   └── ja.json         # 日本語キー
├── sync-menu.ts        # Tauri ネイティブメニューへの翻訳同期
└── translate-error.ts  # Rust エラー文字列 → i18n key のマッピング
```

## Component での使い方

```typescript
import { useTranslation } from 'react-i18next';

const { t } = useTranslation();
// ...
<span>{t('sidebar.newFolder')}</span>
```

## Store / lib/api での使い方

Store では hook が使えないため、`i18n` インスタンスを直接呼ぶ。

```typescript
import i18n from '@/i18n';

toast.error(i18n.t('toast.saveFailed'));
```

## Rust エラーの翻訳

Rust から返ったエラー文字列は `translateError()` を通してから toast に渡す。

```typescript
import { translateError } from '@/i18n/translate-error';

catch (e) {
  toast.error(translateError(String(e)));
}
```

## 新しい文字列を追加するとき

`en.json` と `ja.json` の**両方**に同時に追加する。片方だけ追加しない。

```json
// en.json
{ "toast": { "exportFailed": "Export failed" } }

// ja.json
{ "toast": { "exportFailed": "エクスポートに失敗しました" } }
```
